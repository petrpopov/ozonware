## Design Context

### Users
Владелец небольшого магазина на OZON + команда 2–5 человек (менеджер склада, операторы). Работают за десктопом в офисе или дома, инструмент открыт весь рабочий день. Задачи: приёмка, отгрузка, списание, инвентаризация, синхронизация с OZON FBS/FBO и Google Sheets. Пользователи технически грамотны, но не разработчики — ожидают профессиональный инструмент, не хобби-проект.

### Brand Personality
Три слова: **точный, спокойный, профессиональный**.

Интерфейс — рабочий инструмент, а не витрина. Как хороший нож или профессиональная клавиатура: ничего лишнего, каждая деталь функциональна, в руках ощущается качество. Никакого маркетинга и анимаций ради анимаций — только работа.

### Aesthetic Direction
- **Основа**: Vercel / Geist — ахроматика, shadow-as-border, минимализм. Уже реализовано и сохраняется.
- **Тема по умолчанию**: тёмная. Интерфейс оптимизирован для длительной работы в тёмном режиме.
- **Типографика**: Geist (основной) + Geist Mono (числа, артикулы, коды). Жёсткая иерархия размеров — минимум 1.25× между ступенями.
- **Цвет**: ахроматическая палитра (#0a0a0a base в тёмном режиме), единственный акцент — danger (#ef4444) для деструктивных действий. Статусные цвета (зелёный/красный для дельт) переводятся в CSS-переменные.
- **Антиреференсы**: никакого glassmorphism, gradient text, hero metrics, card grids, bounce-анимаций, border-left accent stripes.

### Design Principles
1. **Информация важнее украшений** — каждый элемент UI существует потому, что несёт смысл. Декоративные элементы без функции удаляются.
2. **Тёмная тема — первичная** — все цвета, контрасты и состояния проектируются под `[data-theme='dark']` в первую очередь.
3. **Числа читаются с первого взгляда** — Geist Mono для всех количеств, цен, ID; фиксированная ширина колонок там, где нужно сравнивать значения.
4. **Спокойные состояния, чёткие сигналы** — нейтральные hover/focus, яркий акцент только на ошибках и деструктивных действиях. Нет ничего, что кричит без причины.
5. **Статусные цвета — в токены** — зелёный/красный/янтарный для дельт и предупреждений должны быть CSS-переменными, а не хардкодом, чтобы корректно работать в обеих темах.
